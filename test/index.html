const 특징

const 는 한번 할당 하면 값이 절대 바뀌지 않는다
값을 선언함과 동시에 값을 절대 변경 할수 없다

  • 값을 계속 변경할수 있는것을 mutable 데이타 타입이라고 한다
    값을 변경할수 없는 걸 Immutable 데이터 타입
  • const 가 좋은 이유 보안상 이유(해커방지)
    thread safety 스레드를 세이프티
    개발자실수방지

array 종류

  • 배열을 문자열로 .join

    const result = fruits.join();
    console.log(result);

    값 : apple/banana/orange

  • 문자열을 배열로 .split

    const fruits = '🍎, 🥝, 🍌, 🍒';
    const result = fruits.split();
    console.log(result);

    const result = fruits.split(',', 2);이렇게 가지고 오고싶은 배열만 가져올수 있음

    값 :  ["🍎", " 🥝"]

  • 배열을 거꾸로 .reverse

    const array = [1, 2, 3, 4, 5];
    const result = array.reverse();
    console.log(result);

    그치만 배열 자체 를 거꾸로 바꿔어 버림

    값 : [5, 4, 3, 2, 1]

  • 배열 삭제 .splice

    const array = [1, 2, 3, 4, 5];
    const result = array.splice(0, 2);
    console.log(result);
    console.log(array);

    값 : 내가 삭제한 배열 [1, 2]
    삭제되어진 배열[3, 4, 5]

  • 배열 삭제 .slice

    const array = [1, 2, 3, 4, 5];
    const result = array.slice(0,2);
    console.log(result);
    console.log(array);

    slice 는 기존 배열에서 내가 선택한 인덱스만 slice 한다

    값 : 내가 삭제한 배열 [1, 2]
    기존 배열[1, 2, 3, 4, 5]

  • 90점 이상인 학생 찾기 콜백 함수

  • true 인 애만 찾기filter

    const result = students.filter((students) => students.enrolled);
    console.log(result);

  • 점수 만 가져오기.map

    const result = students.map((students) => students.score * 2);
    console.log(result);

    .map(value * 2) 하면 점수에다 곱하기 2한 값이 들어간다

  • 50점 이하인 애가 있으면.some

    모든 배열중 하나라도 조건이 충족되면

    배열에서 50점 이하인 애가 있으면 true 없으면 false

    const result = students.some((students) => students.score < 50);
    console.log(result);

    값 : true

  • 50점 이하인 애가 한명이라도 있는지.every

    모든 배열의 조건이 충족

    배열에서 50점 이하인 애가 한명이라도 있으면 fale

    const result = students.every((students) => students.score < 50);
    console.log(result);

    값 : false
